# Infinity Stumps — Status Summary (May 13, 2026)

## Mission

**DRS for the rest of us.** Hawk-Eye-class ball tracking + LBW prediction
at ~1/30th the cost (~£1,500 vs ~£100K+), running on a consumer phone.

## Architecture

**Hardware:**
- 8 Qorvo DWM3001C anchors (4 stump tops, 4 PAI ground)
- One anchor designated **hub stump** — sync master + range aggregator +
  BLE gateway to phone
- Ball: 1× DWM3001C + 1× ADXL372 (event timing, ~£22 silicon)
- iPhone runs all the math via Swift + Accelerate + CoreML
- ~£1,500 total BOM, no edge PC

**Signal chain:**
- 100 Hz broadcast DS-TWR between ball ↔ 8 anchors (ETSI-compliant)
- 7 "dumb" anchors → hub stump over UWB-data-mode
- Hub stump → iPhone via single BLE link
- iPhone: EKF + RTS smoother → trajectory + LBW verdict + ellipse

**Algorithm:**
- Continuous EKF over (position, velocity, spin) using
  drag + Magnus + gravity ODE
- Per-anchor measurement updates handling TWR stagger
- RTS backward smoother for replay-quality precision
- Same code for live overlay (forward only) and DRS replay
  (forward + smoother)

## Architectural decisions log (chronological)

1. **Chip platform** — Qorvo DWM3001C (decision 1 in CLAUDE.md; switched from QPK3000 on 2026-05-14)
2. **Anchor count** — 8 (12 rejected, sim 02 → <2% gain)
3. **Sync** — wireless inter-anchor UWB self-sync at 10 Hz (sim 07b)
4. **Positioning** — broadcast DS-TWR @ 100 Hz (TDoA rejected, sim 07;
   150 Hz rejected, ETSI duty-cycle breach per `airtime-budget.md`)
5. **Ball-side** — DWM3001C + ADXL372 (precision IMU dropped, sim 06)
6. **Analytics** — **EKF + RTS smoother** (batch fit deprecated as
   primary, kept for reference). Locked May 13, 2026.
7. **Topology** — hub stump (1 anchor with extra firmware) +
   iPhone-as-compute. No edge PC.
8. **Spin tracking** — magnetometer-only path planned for v2 (full IMU
   avoided for patent-density reasons).

## What's validated vs what's not

**Validated in simulation:**
- Algorithm architecture (sim_ekf, sim_lbw, sim_realistic)
- Realistic occlusion model (14-bone skeleton batter from
  Drillis-Contini anthropometric ratios + Law-41 bowler trajectory)
- Beats published iBall benchmarks (NSDI '17) on every metric

**NOT yet validated:**
- Real UWB range noise vs 30 mm σ assumption (Phase 1.1 — DWM3001CDK
  boards arriving)
- Real NLOS bias distribution (Phase 2.1 + 2.2)
- Ball shell survivability through cricket impacts (Phase 4a)
- iPhone app + BLE link reliability under field conditions
- Anchor in-field calibration procedure / time
- Battery life through a full session
- Spinning-ball antenna pattern (Phase 2.3)
- Umpire / user acceptance

## Patent posture

**Not filing.** iBall (Gowda et al., NSDI '17, "Bringing IoT to Sports
Analytics") established UWB + physics-fit + LBW-extrapolation for a
cricket ball as public prior art. We **execute** the published idea
with modern silicon, better topology, and outdoor validation. The
project value is execution speed + product quality + iPhone-native UX,
not IP.

Magnetometer-only spin path (v2) — avoids full-IMU patent density from
Catapult / STATSports / Zepp / etc.

See `docs/prior-art.md` for the full breakdown.
